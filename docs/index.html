// THE GAME

한 줄로 끝나지 않는 루프

NEON DRIFT는 채집 → 전투 → 강화가 맞물려 돌아가는 아케이드 웨이브 디펜스다. 준비 단계에서 호버바이크를 몰아 필드의 자원 블록을 부수고, 전투 단계에서 기지 포탑에 올라 밀려오는 적을 막는다. 살아남은 자원으로 기체와 포탑을 강화하며 5웨이브 끝까지 버틴다.

// CORE MECHANICS

핵심 메커닉

✦

호버바이크 비행

중력 없는 6DOF 이동. 지수 감속(드래그)과 자동 호버로 미끄러지는 관성감을 구현했다. 속도에 따라 FOV가 70°→110°로 보간돼 가속감을 강조한다.

◎

자석 자원 채집

블록을 파괴하면 부스러기가 떨어지고, 자석 범위 안에 들어오면 호밍 가속으로 빨려든다. 강화로 자석 반경이 늘어난다.

▣

공격력 게이트 블록

Gold·Iron·Diamond 3종 자원 블록. 기체 공격력이 블록의 RequiredPower에 못 미치면 스파크만 튀고 파괴되지 않는다 — 강화를 강제하는 진행 장치.

⊕

포탑 탑승 전투

E 키로 기지 수동 포탑에 탑승. 회전 속도가 제한돼 조준에 무게감이 있다. 카메라/포신 두 갈래 조준점으로 탄착을 읽는다.

⟁

웨이브 디펜스

5웨이브 5/10/15/20/30마리. 후반엔 4방향 입구에서 동시 스폰. 적은 기지에 도달하면 원거리에서 지속 피해를 가한다.

↟

강화 상점

기체와 포탑 통합 8종 강화 — 자석·공격력·속도·HP·포탑 회전·연사·자동포탑 수. 웨이브 사이 자원으로 단계별 구매.

// UNDER THE HOOD

기술 스택 & 개발 정보

에디터에서 만드는 에셋을 최소화하고 게임플레이 전체를 C++로 작성하는 것을 설계 원칙으로 삼았다. 입력·UI·머티리얼 제어까지 코드로 다뤄 CLI 빌드만으로 검증 가능한 구조를 만들었다.

Enhanced Input — 코드 생성

InputMappingContext / InputAction을 .uasset 없이 PlayerController에서 NewObject + MapKey로 구성. 입력 에셋 0개.

Canvas HUD — 순수 C++

UMG 위젯 없이 AHUD::DrawHUD() Canvas 드로잉으로 자원·HP 바·웨이브·상점·포탑 조준점까지 전부 구현.

상태머신 게임 루프

PreWave→Gather→Combat→Shop→Victory/GameOver 7상태 전이를 GameMode가 관리. 웨이브 스폰 펌프와 자원 카운트 통합.

IDamageable 인터페이스

블록·몬스터·기지가 공통 TakeHit(Damage, AttackerPower)를 구현. 라인트레이스가 맞은 액터에 인터페이스 캐스트로 피해 전달.

Dynamic Material Instance

네온 부모 머티리얼 1개만 수동 제작. 색·발광·피격 플래시는 전부 C++ MID 파라미터로 런타임 제어.

Niagara 트레일 & 폴리시

기체·몬스터 파티클 트레일, 카메라 셰이크, 네온 펄스 등 시각 피드백을 코드 측에서 연결.
